PURPOSE
═══════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════

V13 drives InvestorClaw end-to-end by SIMULATING AN END USER against each deployed agent CLI. Claude is the robot user. The harness
NEVER executes investorclaw.py directly. Every test invokes an agent (OpenClaw | ZeroClaw | Hermes) via its CLI, the agent routes the
invocation through the InvestorClaw skill, and Claude reads the agent response, validates the ic_result envelope, and records exact
evidence. Fabrication is forbidden. Every test produces a raw transcript on disk; summary-only reporting is a VIOLATION.

This spec is the authoritative runbook. It REPLACES the Python harness v13_linux-x86-host_enterprise_cicd.py (retired). There is no Python
execution of this matrix. Claude executes each step directly via Bash or Agent CLI invocations as instructed.

BLOCKER HANDLING
═══════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════

ENVIRONMENTAL (retry ≤ 2 per test)
  Typical fixes: openclaw gateway restart | zeroclaw daemon restart (ssh + systemctl) | export missing env var | free memory |
  reopen SSH | re-issue /new to session.
  After 2 failed retries: surface as TEST_UNRESOLVED and continue matrix. Do NOT fabricate success.

SKILL_CODE_DEFECT (mandatory 6 steps — identical to V6.1.2)
  1. Agent reports error + suggested fix (log verbatim as blocker_detail.agent_report).
  2. Claude verifies (reproduce via same invocation or surgical repro on linux-x86-host; log verification.method).
  3. Claude edits source under the repo directly.
  4. git add <files> && git commit → log nas_repair_commit_sha.
  5. git push origin <branch> → log github_repair_push_confirmed.
  6. Restart W0 for the affected session (skill reinstall on remote host if that host was involved), increment restart_count.
  Re-run the failing test. If it fails again, cycle once more with a distinct repair commit; second consecutive failure under
  the same defect class = BLOCKED, surface and continue matrix.

OPENCLAW_SOURCE_FLAW
  1. Agent reports flaw in OpenClaw itself (not InvestorClaw).
  2. Claude verifies flaw is outside InvestorClaw repo.
  3. File upstream PR to https://github.com/openclaw/openclaw (log openclaw_pr_branch + openclaw_pr_url).
  4. Block the affected config but continue the rest of the matrix. Log blocker_detail.pr_url.

ZEROCLAW_DAEMON_FLAW / HERMES_DAEMON_FLAW
  1. Agent daemon returns error outside skill boundary.
  2. Verify via ssh <host> systemctl status <daemon>.service (zeroclaw.service, hermes.service) + log output.
  3. Restart daemon (ssh <host> sudo systemctl restart <daemon>.service). If persistent, file upstream PR, block affected sessions,
     continue.

NEMOCLAW_SANDBOX_VIOLATION
  1. Occurs on pi-small only (openshell capability/rlimit refusal).
  2. Verify: is this the sandbox correctly blocking something (intended) or a genuine flaw?
  3. If intended: log as expected constraint, skip that specific test (log SKIP_REASON=SANDBOX_BY_DESIGN).
  4. If flaw: file upstream PR to nclawzero, block pi-small configs if pervasive, continue.

CONSULTATION_ENDPOINT_UNREACHABLE
  1. Hybrid configs (TG-H, GM-H) require gpu-host 192.0.2.96:8080 to be reachable from the execution host.
  2. Pre-flight probe (PF3 below) determines reachability per host. If host X cannot reach gpu-host, all hybrid tests scheduled
     for host X are SKIPPED with SKIP_REASON=CONSULTATION_ENDPOINT_UNREACHABLE. NOT a failure — a documented skip.
  3. Cloud configs (TG-C, GM-C) do not depend on gpu-host and proceed.

MODEL_UNAVAILABLE
  1. Narrative or consultation model refused (404, "model not found", gate rejected).
  2. Verify key present in OpenClaw .env on linux-x86-host (and replicated to the relevant host).
  3. Cross-check model id against MODELS.md (the only authoritative source of supported model strings).
  4. If key present but model offline: provider outage, skip that config, continue, flag provider_instability.

UNFIXABLE
  Log → surface at end-of-run → no repair attempted. Test record: alert_level=fail, blocker_type=unfixable.

Provider registration pattern (applies to all agents): each agent discovers a provider via (provider_id, api_base, api_key,
model_catalog). OpenClaw declares providers in ~/.openclaw/openclaw.json::models.providers; ZeroClaw via
~/.zeroclaw/config.toml::providers; Hermes via ~/.hermes/config.yaml::inference. TogetherAI on Hermes uses the "custom provider"
pattern: provider=openai, api_base=https://api.together.xyz/v1, api_key=<TOGETHER_API_KEY> — same trick OpenClaw uses for any
OpenAI-compatible endpoint. This is documented in v6.1.2 and OpenClaw's provider docs as the canonical way to register
non-first-class endpoints.

             (Hermes uses config.yaml inference block pinned in W0.3.2; no per-invocation provider/model override to avoid

═══════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════
EXECUTION CONTRACT (per test)
═══════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════

FOR EACH session S in [S01 .. S20]:
  IF pre-flight marked S as SKIP with SKIP_REASON: emit per-session skip record and continue to next session.
  EXECUTE W0.1 → W0.5 for S.
  IF W0 fails: emit session-blocker record, skip all W/N tests for S, continue.
  FOR EACH test T in [W1..W22, N001..N250]:
    1. COMPOSE exact agent-CLI command per S.agent and T prompt template.
    2. EXECUTE (Bash via ssh + agent CLI). Capture stdout, stderr, exit code, elapsed_ms, start_ts_utc.
    3. EXTRACT ic_result envelope (jq '.ic_result // empty' on the last JSON line of stdout OR equivalent grep+parse in Bash).
    4. EVALUATE:
       - exit_code expected = 0 (except W21 /help on some agents may echo exit 0 directly; still expect ic_result)
       - ic_result present = REQUIRED for slash-command tests (W1..W22)
       - ic_result present = REQUIRED for NL tests that route to a skill command (agent decides routing; if agent answers from
         model knowledge with no skill invocation, ic_result may be absent; log agent_routed_to_skill=false, mark UNVERIFIED
         for skill-coverage purposes but PASS for conversational coverage)
       - no FABRICATION phrases in agent response
       - no leaked keys or PII
    5. APPEND per-test transcript to <run-dir>/transcripts/<session_id>/<test_id>.json
    6. EMIT the 5-section RESPONSE FORMAT block (🎯 📝 ✅ 📊 🔍) to the running log.

INTER-TEST PAUSE: 500ms minimum between invocations to avoid provider burst throttling.
RATE-LIMIT BACKOFF: on 429, apply (2s, 4s, 8s, ±10% jitter, cap 120s, max 5 retries). Log each retry. Exhausted → log
  RATE_LIMIT_VIOLATION=false (429 correctly handled, exhausted = provider unavailability), test alert=fail blocker=rate_limited.
